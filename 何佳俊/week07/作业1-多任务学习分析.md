## 2. 多任务训练 loss = seq_loss + token_loss 的坏处

### 问题
1. **量级不平衡**：
   - seq_loss 是句子级别，每个样本一个 loss 值
   - token_loss 是 token 级别，每个样本有多个 loss
   - 直接相加可能导致某个任务主导梯度更新

2. **收敛速度不同**：
   - 如果一个任务学习得快，另一个学得慢，简单的加法会导致训练不平衡
   - 快的任务可能过拟合，慢的任务欠拟合

3. **梯度冲突**：
   - 两个任务的梯度方向可能不一致，相互抵消

### 解决方案

#### 方案 1: 加权求和
```python
loss = alpha * seq_loss + beta * token_loss
# 根据经验调整权重，如 alpha=0.5, beta=0.5
```

#### 方案 2: 动态权重（GradNorm）
根据梯度范数自动调整权重：
```python
# 计算每个任务的梯度范数
seq_grad_norm = torch.norm(torch.autograd.grad(seq_loss, shared_params, retain_graph=True)[0])
token_grad_norm = torch.norm(torch.autograd.grad(token_loss, shared_params, retain_graph=True)[0])

# 动态调整权重使梯度平衡
total_grad_norm = seq_grad_norm + token_grad_norm
alpha = total_grad_norm / (2 * seq_grad_norm)
beta = total_grad_norm / (2 * token_grad_norm)
loss = alpha * seq_loss + beta * token_loss
```

#### 方案 3: Uncertainty Weighting（推荐）
为每个任务学习一个不确定性参数：
```python
import torch.nn as nn

class MultiTaskLoss(nn.Module):
    def __init__(self):
        super().__init__()
        # 可学习的对数方差参数
        self.log_var_seq = nn.Parameter(torch.zeros(1))
        self.log_var_token = nn.Parameter(torch.zeros(1))
    
    def forward(self, seq_loss, token_loss):
        # 基于不确定性的加权
        loss_seq = seq_loss / torch.exp(self.log_var_seq) + self.log_var_seq
        loss_token = token_loss / torch.exp(self.log_var_token) + self.log_var_token
        return loss_seq + loss_token
```

#### 方案 4: 归一化 Loss
将两个 loss 归一化到相同量级：
```python
# 使用移动平均记录历史 loss 值
seq_loss_avg = moving_average(seq_loss)
token_loss_avg = moving_average(token_loss)

# 归一化
norm_seq_loss = seq_loss / (seq_loss_avg + 1e-6)
norm_token_loss = token_loss / (token_loss_avg + 1e-6)

loss = norm_seq_loss + norm_token_loss
```
