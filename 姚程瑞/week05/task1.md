# 客服工作台

作为后端开发与算法开发，需为客服工作台的 FAQ 类目管理、录入管理及智能答疑核心场景搭建技术体系，实现机器替代人工客服的核心能力，从数据存储、模型应用、AI 能力落地等维度完成技术支撑，以下是核心工作内容总结：

1. **数据库设计**：必须设计专属数据库，需拆分多表实现精细化存储。包含类目表（存储一级 / 无限级类目 ID、名称、层级关系、父级 ID 等）、FAQ 主表（FAQID、标题、答案、类目关联 ID、生效时间、环境状态、修改记录等）、FAQ 扩展表（相似问法、关联问题、标签、答案视角 / 类型等），同时为类目检索、FAQ 关键词 / ID 搜索设计索引，保障查询效率，支撑测试 / 正式双环境的数据隔离与同步。

2. **模型选择**：核心选用**BERT 模型**作为文本相似度匹配的基础 AI 模型，搭配基础 NLP 分词、文本预处理工具完成前置处理；大模型为可选补充模型，非强制使用，可根据业务需求灵活集成。

3. **BERT 的使用方式**：将用户提问、FAQ 标准标题、相似问法均进行统一预处理后，输入 BERT 模型做语义编码，转化为等维度的语义向量；通过计算用户提问向量与历史 FAQ 向量的相似度，筛选出最匹配的 FAQ 答案；同时可利用 BERT 对新增 FAQ 的相似问法做语义校验，避免重复录入，提升匹配精准度。

4. 大模型的使用考量

   ：无需强依赖大模型，但可作为 AI 能力升级的补充。例如利用大模型自动生成 FAQ 的相似问法，提升运营录入效率；对用户口语化、模糊化提问做语义规整，再交由 BERT 编码匹配；也可借助大模型实现 FAQ 多视角答案的自动生成、答案内容的智能优化，进一步提升智能答疑的体验。

   

   此外，后端开发需实现类目 / FAQ 的 CRUD 接口、双环境数据管理、批量导入 / 导出 / 删除等功能接口，封装算法服务调用；算法开发负责 BERT 模型的部署、调优、文本编码与相似度计算的算法服务开发，以及可选的大模型能力集成，实现业务与 AI 技术的协同落地。
