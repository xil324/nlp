### 2. 主要内容概述

#### 标题与作者信息

#### 摘要（Abstract）
- **背景问题**：
  - RAG 技术虽能有效帮助 LLM 从外部知识源中检索信息以回答具体问题，但难以应对**全局性、概括性问题**（例如：“这个数据集的主要主题是什么？”），这类任务属于**查询聚焦摘要（QFS）**，而非简单的检索任务。
  - 传统的 QFS 方法无法扩展到大型文档集合（尤其是百万级 token 范围内）。

- **解决方案**：
  - 提出 **GraphRAG**：一种基于图的问答方法，适用于私有文本语料库。
  - 该方法具有可扩展性，能够适应不同复杂度的用户查询和大量源文本。
  - 实现流程分为两阶段：
    1. 使用 LLM 构建一个**实体知识图谱**（entity knowledge graph），从原始文档中提取实体及其关系。
    2. 对图中的每个紧密相关的实体群组（community）生成**社区摘要**（community summaries）。
  - 当收到查询时：
    - 首先利用这些社区摘要生成部分响应；
    - 再将所有部分响应汇总成最终答案。

- **实验结果**：
  - 在包含约 100 万 token 的数据集上测试了针对全球性语义问题的回答性能。
  - 相比传统 RAG 基线方法，GraphRAG 在**答案的全面性和多样性**方面表现出显著改进。

#### 引言（Introduction）
- 简要回顾了 RAG 的基本原理：当输入数据超过语言模型上下文窗口限制时，RAG 通过检索相关片段来辅助生成。
- 指出现有 RAG 方法在处理**全局性、高层次总结类问题**时存在局限性。
- 引入 GraphRAG 的动机：结合图结构建模的优势与 LLM 的生成能力，实现更高效、更准确的全局问答。
