# DeepSeek-V3 相比 V2 的主要改进

DeepSeek-V3 并非简单放大 V2，而是在继承 V2 已验证架构的基础上，通过多项关键创新实现了质的飞跃。以下从六个维度系统总结 V3 相较于 V2 的核心改进。

---

## 一、模型规模跃升

V3 在保持经济高效原则的前提下，大幅扩展了模型容量。

| 对比维度          | DeepSeek-V2         | DeepSeek-V3         | 提升幅度        |
| ------------- | ------------------- | ------------------- | ----------- |
| 总参数           | 236B                | 671B                | **约 2.8 倍** |
| 激活参数（每 Token） | 21B                 | 37B                 | 约 1.8 倍     |
| 稀疏程度（总参/激活参）  | 约 11.2 倍            | 约 18.1 倍            | **稀疏性大幅提升** |
| 专家总数          | 160 个路由专家 + 2 个共享专家 | 256 个路由专家 + 1 个共享专家 | **专家更细粒度**  |
| 每 Token 激活专家数 | Top-6               | Top-8               | 更多专家参与      |
| 路由评分函数        | Softmax             | Sigmoid             | 路由更稳定       |

V3 的总参数是 V2 的近三倍，但激活参数仅增加到 1.8 倍，稀疏程度从约 11 倍跃升至约 18 倍，意味着在推理时**激活更少的参数比例，获得了更强的表达能力**。同时，每 Token 激活的专家数从 6 个增加到 8 个，路由机制也从 Softmax 改为 Sigmoid，有效降低了路由决策的波动性。

V3 的模型架构延续了 V2 的核心设计——按层数来看，前 3 层采用密集层（Dense Layer），其余 58 层为 MoE 层。hidden_size 从 5120 提升至 7168，大大增强了每一层的表示能力。

---

## 二、MoE 负载均衡创新：从辅助损失到无辅助损失

这是 V3 在 MoE 架构层面最核心的突破之一，也是**相较于 V2 的最大区别**。

- **V2 的做法**：沿用传统 MoE 的辅助损失来强制专家负载均衡，但这种机制会干扰模型学习，在一定程度上以牺牲性能为代价换取负载均衡。
- **V3 的创新机制**：在 V2 基础上实现了关键突破——**首次在超大规模 MoE 训练中摆脱了辅助损失的束缚**，设计了**不依赖辅助损失的解决方案**：
  - 为每个专家引入一个**动态偏置项**，在训练过程中根据负载情况实时调整。
  - 偏置项仅用于路由选择，不影响门控值（即模型输出计算的权重）。
  - 负载过高的专家：偏置减少，降低被选中概率；负载不足的专家：偏置增加，提高被选中概率。
- 整套机制只控制**哪条路径走通**，但不干扰模型**对走通路径的计算结果**，从而在实现负载均衡的同时**保留了模型表达的纯净性**，相比使用辅助损失显著提升了模型性能。

---

## 三、训练目标革新：从单 Token 到多 Token 预测

- **V2**：采用传统的单 Token 预测（Next Token Prediction），模型在每个位置仅预测下一个 Token。训练信号密度较低，尤其在长文本场景下难以高效学习长期依赖。

- **V3**：首次引入了 **多 Token 预测目标（Multi-Token Prediction, MTP）**，将预测范围扩展到每个位置的多个后续 Token（即“走一步看三步”）。多阶段 Transformer 模块的设计保持了完整的因果链，且不同预测深度共享词嵌入与输出头。在推理阶段，这些 MTP 模块可用于推测解码，生成速度提升约 1.8 倍，且第二个 Token 预测接受率高达 85%–90%。

---

## 四、FP8 混合精度训练：从 BF16/FP16 到 FP8

V3 是**全球首个在万亿参数级 MoE 模型上成功实现 FP8 混合精度训练**的模型，V2 则仍采用 BF16/FP16 精度。

| 对比维度  | DeepSeek-V2 | DeepSeek-V3                            |
| ----- | ----------- | -------------------------------------- |
| 训练精度  | BF16 / FP16 | **FP8（E4M3 格式）**                       |
| 细粒度量化 | 无           | per-tile (1×128) + per-block (128×128) |
| 高精度累加 | 常规          | Tensor Core + CUDA Core 混合累加           |
| 显存占用  | 基准          | 相比 FP16 降低约 35%                        |
| 训练稳定性 | 较稳定         | 损失相对误差 < 0.25%                         |

V3 通过 FP8 训练实现了**同等硬件下显存占用大幅降低**。

---

## 五、训练成本与效率：持续优化的极致性价比

| 对比维度           | DeepSeek-V2     | DeepSeek-V3     | 变化           |
| -------------- | --------------- | --------------- | ------------ |
| 预训练数据量         | 8.1 万亿 Token    | 14.8 万亿 Token   | **增加 82.7%** |
| 训练算力成本         | 约 276 万美元（推测）   | **约 557.6 万美元** | 仅增加约 1 倍     |
| 每万亿 Token 训练成本 | 约 172.8K GPU 小时 | 约 180K GPU 小时   | 几乎持平         |
| 训练稳定性          | 较好              | 全程无损失崩溃、无回滚、无重启 | **更稳定**      |

V3 的训练数据量比 V2 增加接近一倍，而训练成本仅增加约 1 倍。得益于 MoE、MLA、无辅助损失均衡、MTP 以及 FP8 混合精度训练的协同作用，V3 在参数翻倍的情况下保持了近乎线性的成本增长，且训练过程极其稳定。

---

## 六、推理性能飞跃：速度翻倍，延迟更低

| 对比维度       | DeepSeek-V2 | DeepSeek-V3             |
| ---------- | ----------- | ----------------------- |
| 生成吐字速度     | 约 20 TPS    | **提升至 60 TPS（3 倍提升）**   |
| 首 Token 延迟 | 约 189ms     | **约 127ms（降低 32%）**     |
| 推理吞吐量      | 基准          | **动态批处理后提升 2.1 倍**      |
| 专家并行规模     | 较小          | EP320 + 冗余专家技术          |
| 通信方式       | All-to-All  | 点对点 IB 传输 + IBGDA 低延迟优化 |
| 负载均衡       | 传统辅助损失      | 动态偏置 + 冗余专家部署           |

V3 的推理改进得益于**专家并行规模的极致放大**和**冗余专家动态部署技术**。在解码阶段，最小部署单元达到 40 个节点（320 张 GPU），采用 320 路专家并行，理想状态下每张 GPU 仅需负责 1 个专家。

---

## 总结

- **宏观层面**：V3 将稀疏程度的效率性价比推向了新高度——总参 671B，激活仅 37B，稀疏度从约 11 倍扩大到约 18 倍，保证了极低的推理计算成本。

- **架构层面**：V3 在继承 V2 的 MLA 和 DeepSeekMoE 核心理念的基础上，实现了两大革命性突破——**无辅助损失负载均衡**（彻底解决了传统 MoE 中性能与负载均衡不可兼得的困境）和**多 Token 预测**（从“走一步看一步”升级为“走一步看三步”）。

- **工程层面**：V3 成为全球首个在万亿参数 MoE 上成功使用 FP8 混合精度训练的模型，并在训练过程中实现了全程无损失崩溃、无回滚的极高稳定性。

- **性能层面**：V3 在代码、数学、中英文理解上全面对标 GPT-4o 和 Claude-3.5-Sonnet，同时**首 Token 延迟降低 32%，吞吐量提升 2.1 倍**，成为开源领域中性价比与性能的双重标杆。
