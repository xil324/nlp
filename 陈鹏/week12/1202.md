# 阅读 06-stock-bi-agent 代码，回答如下问题：

## 什么是前后端分离？
    前端负责页面展示，
    后端负责数据接口，
    两者通过 API 通信，独立开发、独立部署。

## 历史对话如何存储，以及如何将历史对话作为大模型的下一次输入

    历史对话是 通过append_message2db 存在 sever.db的 ChatSessionTable、 ChatMessageTable 两个表中
    
    每次通过下面这个，把历史对话存起来，作为大模型的下一次输入
    session = AdvancedSQLiteSession(
        session_id=session_id, # 与 系统中的对话id 关联，存储在关系型数据库中
        db_path="./assert/conversations.db",
        create_tables=True
    )
    
    Runner.run_streamed(agent, input=content, session=session)
